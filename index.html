Key Innovation: Personalization

A `/remember` endpoint allows the system to store user-specific facts (e.g., name, role). This information is injected into the prompt, transforming the generic bot into a personalized assistant that can tailor responses to the user's context.

Key Innovation: CSV Analysis

The system includes a dedicated tool for analyzing structured data from CSV files. Instead of embedding the CSV, it passes the content directly to the LLM with a specialized prompt for analytical tasks, effectively turning the RAG system into a reasoning engine.

Strategy 2: Metadata-Enriched Retrieval

This strategy enhances the data *before* it's queried, using an LLM to add rich metadata to each text chunk, thereby improving retrieval precision.

The Core Trade-Off: Cost of Quality

This approach significantly increases retrieval precision but at a high cost. The indexing process becomes much slower and more computationally expensive due to the multiple LLM calls required for every single text chunk.

Strategy 3: GraphRAG for Interconnected Knowledge

This advanced strategy moves beyond simple retrieval to model the entire knowledge base as a graph, enabling the discovery of deep, distributed insights.

Hybrid Retrieval Process

1. Query Received
User submits a question.
‚Üì
2. Vector Search for Entry Points
A standard vector search identifies the most relevant nodes to start from.
‚Üì
3. Graph Traversal for Context Expansion
The system "walks" the graph from the entry points, collecting connected information.
‚Üì
4. Synthesize Rich Context
The collected subgraph of knowledge forms a rich, interconnected context.
‚Üì
5. LLM Generation
The rich context is passed to the LLM to generate a comprehensive answer.

Anatomy of the Graph

The knowledge graph is built with text chunks as nodes. Edges (connections) are created based on a sophisticated dual-criteria system:

  • Semantic Similarity: Connects chunks with very similar vector embeddings.
  • Shared Concepts: Connects chunks that share named entities (like people, places) or key concepts extracted by an LLM.

The Risk: Contextual Drift

While powerful, traversing the graph can sometimes lead to irrelevant information. A key challenge is tuning the traversal depth: too shallow, and you miss context; too deep, and you introduce noise that can confuse the LLM.
